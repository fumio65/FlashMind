## Terminology / Acronyms

| Term | Meaning |
|---|---|
| FBA | Feature-Based Architecture — code organized by domain, not file type |
| Deck | A named collection of flashcards grouped by subject |
| Card | A single flashcard — `front` (term) + `back` (definition) + optional images |
| Flashcard Mode | Study mode: flip card → mark Known or Still Learning |
| Quiz Mode | Study mode: auto-generated MCQ with 30s countdown timer |
| Mastery % | (Known cards / total cards) × 100 for a given deck |
| Study Streak | Consecutive days with ≥1 completed study session |
| Known | Card the user marked as Got It! (→ arrow or button) |
| Still Learning | Card the user marked as needs more review (← arrow or button) |
| Creator | Student role that publishes public decks |
| shadcn/ui | Copy-paste UI library built on Radix UI + Tailwind CSS — components live in `src/components/ui/` |
| `cn()` | Utility from `src/lib/utils.js` — merges Tailwind class names safely (clsx + tailwind-merge) |
| Barrel file | `index.js` in each feature folder that re-exports only what's needed externally |

## Naming Conventions

| Type | Convention | Example |
|---|---|---|
| Components | PascalCase | `DeckCard.jsx`, `FlipCard.jsx` |
| Hooks | camelCase + `use` prefix | `useDecks.js`, `useLogin.js` |
| API modules | camelCase + `.api` suffix | `decks.api.js`, `auth.api.js` |
| Zustand stores | camelCase + `Store` suffix | `authStore.js` |
| Pages | PascalCase + `Page` suffix | `DashboardPage.jsx` |
| CSS classes | Tailwind utility only — no custom CSS classes unless necessary | `className="flex gap-4 p-6"` |
| Git branches | kebab-case with type prefix | `feat/auth`, `feat/decks`, `fix/quiz-timer` |
| Commits | Conventional Commits | `feat(auth): add JWT login` |

## File Structure Rules
- **Pages are thin** — they only compose feature components. Zero business logic in page files.
- **Features are self-contained** — each feature has its own `components/`, `hooks/`, `api/`, and `index.js`
- **Shared components** (`src/components/`) never import from features
- **shadcn/ui components** live in `src/components/ui/` — generated by CLI, then customized as needed
- **All Axios calls** go through `src/lib/axios.js` — never use `fetch()` directly
